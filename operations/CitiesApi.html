apiCitiesTrendingDestinationsGet

This endpoint provides a list of the most visited cities, serving as a curated selection of trending travel destinations. Each entry in the response includes key details such as: - The city's unique identifier. - The city's name. - A visually appealing thumbnail image. The number of destinations retrieved can be adjusted by specifying the count parameter. If the parameter is omitted, the default value of 5 is used.

URL

//api/Cities/trending-destinations

HTTP Method

GET

Response Type

List

Parameters

  • Query: count Integer

    The number of trending destinations to retrieve. Defaults to 5 if no value is provided.

getCities

This endpoint allows clients to fetch cities with options for sorting, pagination, and filtering. The available query parameters include: - **SortOrder**: Specifies the sorting order (e.g., ascending or descending). - **SortColumn**: Specifies the column to sort by (e.g., name, population). - **PageNumber**: Specifies the page number for pagination. - **PageSize**: Specifies the number of items per page. - **SearchTerm**: Filters cities based on a search term that matches city names. Example usage: GET /cities?sortOrder=asc&sortColumn=name&pageNumber=1&pageSize=10&searchTerm=paris The response includes paginated metadata in the `X-Pagination` header and links to related pages (e.g., next, previous).

URL

//api/Cities

HTTP Method

GET

Response Type

List

Parameters

  • Query: PageNumber Integer

  • Query: PageSize Integer

  • Query: SearchTerm String

  • Query: SortColumn CitySortColumn

  • Query: SortOrder SortOrder
